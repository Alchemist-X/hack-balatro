## 新 TOP 3（立即执行）

### 1. StateEncoder 刷新 + Reward Shaping（B-16 + B-17 合并）

**为什么最优先**：StateEncoder 是训练的硬性前提。当前 454-dim obs 看不到 enhancement、edition、seal、voucher、boss effect、consumable、open pack 等信息。agent 对这些系统完全是盲的。

**具体目标**：
- 更新 `env/state_encoder.py`，加入：
  - Card enhancement (8 types one-hot per card slot)
  - Card edition (4 types one-hot per card slot)
  - Card seal (4 types one-hot per card slot)
  - Consumable inventory (2 slots × type encoding)
  - Voucher ownership (10-dim binary)
  - Boss effect indicator (28-dim one-hot)
  - Open pack state (type + picks remaining)
  - Joker runtime_state summary (scaling values)
- Reward shaping：
  - Blind clear: +1.0
  - Boss clear: +3.0
  - Game win: +10.0
  - Game loss: -1.0
  - Money delta: +0.01 per $1 gained
  - Score efficiency: bonus for beating blind with fewer hands

### 2. PPO Training Smoke Test（B-20）

**为什么优先**：不跑一次完整训练循环，不知道管道是否真的能工作。

**具体目标**：
- 写一个 `scripts/train_smoke.py`，跑 100 个 PPO update
- 验证：env.reset → obs → model.forward → action → env.step → reward → rollout → PPO.update
- 不求 agent 变强，只求管道不报错
- 输出：loss 曲线、平均 episode reward、平均 ante reached

### 3. 更强的 Rule-Based Agent（B-21）

**为什么优先**：
- 当前 agent 连 Ante 5 都打不到，说明策略太弱
- 一个能通关的 rule agent 可以：验证引擎完整性、生成 BC 数据、设定有意义的 baseline
- 即使 RL 训练开始了，也需要一个 baseline 来衡量进步

**具体目标**：
- 升级 `agents/simple_rule_agent.py` 或新建 `agents/smart_rule_agent.py`
- 加入：
  - 手牌评估：考虑 Joker synergy 和 Enhancement 加成
  - Shop 决策：优先买高价值 Joker、有用 Voucher、升级 Planet
  - 弃牌策略：弃掉不配合当前 Joker 的牌
  - 经济管理：维持 $5 倍数以赚利息
  - Boss 应对：根据 Boss 类型调整出牌策略
- 目标：至少在部分 seed 上能通关（Ante 8）
