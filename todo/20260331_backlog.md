## TOP 3（下一步立即执行）

### 1. 正式 PPO 训练实验（100K+ steps）

smoke_test_v1 只跑了 10K steps，agent 仍在随机探索。需要跑一个足够长的实验来观察学习信号。

**具体目标**：
- 100K steps 基线实验 → 观察 mean_ante 是否上升
- 调 `entropy_coef` 从 0.01 → 0.05（防止 entropy collapse）
- 调 reward scale（当前 blind clear +1 可能太小，win +10 太远）
- 目标：agent 稳定过 Ante 2

**判断标准**：如果 100K steps 后 mean_ante 仍为 1.0，说明 reward/obs/action mask 有问题，需要诊断。

### 2. Vectorized Env（提升训练吞吐）

当前 env 采集占 95% 训练时间（373 sps 中 model forward 很快，瓶颈是 Python for 循环逐个 env step）。

**具体目标**：
- 方案 A：在 Rust 侧实现 `VecEngine`，一次 step N 个 engine，通过 PyO3 暴露
- 方案 B：用 Python multiprocessing + SubprocVecEnv 模式
- 目标吞吐：2000+ steps/sec（当前 373）

**优先方案 A**：Rust 侧 batch 无 GIL 问题，预期 5-10x 提升。

### 3. 真实客户端 Trajectory 录制

所有验证仍是 engine 自对照。这是项目最大的战略风险。

**具体目标**：
- 打通 Steamodded + Lovely + BalatroBot 链路
- 录制 1 条完整的真实 trajectory（至少 5 个 blind）
- 用 `diff_replays.py` 对比 engine 和真实客户端的 snapshot
