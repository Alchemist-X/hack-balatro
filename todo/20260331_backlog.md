# hack-balatro Backlog（2026-03-31）

## 路线转向：Small LM + Long Horizon Planning

### 为什么放弃端到端 PPO

6 次 PPO 实验（930K steps）证明：vanilla PPO 无法在 Balatro 的 toggle-based action space 上冷启动学习。Agent 在 select/play 多步序列上完全卡住。同时，一个零训练的 LLM 策略（手写的手牌评估 + 贪心选择）就能过 Small + Big Blind。

**核心洞察**：Balatro 是一个需要推理的游戏，不是一个需要反射的游戏。选牌、经济管理、Joker 组合、Boss 应对——这些都是 planning 问题，不是 pattern matching 问题。

### 新方向：小参数量语言模型（~23B）

用一个参数量较小的语言模型（~23B 级别）作为 agent 大脑。这类模型：
- 推理能力相对较弱，但可以通过 fine-tuning 习得 Balatro 领域知识
- 适合做 Long Horizon Reasoning and Planning 的研究载体
- 推理成本可控，可以大量采样和迭代
- 可以输出 chain-of-thought，决策过程可解释

## 训练计划

### Phase 0: 数据准备（当前阶段）

**目标**：构建高质量的 (game_state, reasoning, action) 数据集

| 步骤 | 任务 | 说明 |
|------|------|------|
| 0.1 | **Game state → text 序列化** | 把 engine snapshot 转成结构化自然语言描述（手牌、Joker、经济、Boss、目标分数） |
| 0.2 | **强 LLM 生成示范数据** | 用 Claude API 玩 100+ 局，每步输出 CoT reasoning + action |
| 0.3 | **数据格式定义** | 定义 SFT 训练数据格式：`<state> <think> <action>` |
| 0.4 | **质量筛选** | 只保留过了 Ante 4+ 的 trajectory；标注关键决策点（shop 选择、boss 应对） |

### Phase 1: Supervised Fine-Tuning (SFT)

**目标**：让 23B 模型习得 Balatro 的基本决策能力

| 步骤 | 任务 | 说明 |
|------|------|------|
| 1.1 | **选择基座模型** | 候选：Qwen2.5-32B、Llama-3-8B/70B、Mistral-22B、DeepSeek 系列 |
| 1.2 | **SFT 训练** | 在 Phase 0 收集的 (state, reasoning, action) 数据上做 LoRA / full fine-tuning |
| 1.3 | **Evaluation** | 让 SFT 模型自主玩 100 局，统计 mean ante / win rate |
| 1.4 | **Error analysis** | 分析失败局的关键错误：是不会选牌？不会买 Joker？不会应对 Boss？ |

**成功标准**：SFT 模型 mean ante >= 3.0（超过 simple_rule_agent 的 Ante 4 水平之前的版本）

### Phase 2: RL Fine-Tuning（强化学习 + 自我博弈）

**目标**：在 SFT 基础上用环境反馈进一步提升

| 步骤 | 任务 | 说明 |
|------|------|------|
| 2.1 | **Rejection Sampling / Best-of-N** | 每步生成 N 个 action，选 reward 最高的。最简单的 RL 信号 |
| 2.2 | **GRPO / DPO** | 用 (winning trajectory, losing trajectory) 对比对做偏好学习 |
| 2.3 | **Online RL (PPO on LM)** | 用 engine reward 信号直接对 LM 做 PPO fine-tuning |
| 2.4 | **Iterated self-play** | 生成 → 筛选 → 训练 → 生成，循环迭代 |

**成功标准**：RL 模型 win rate >= 30%（Stake 1）

### Phase 3: Long Horizon Planning

**目标**：让模型具备跨多步的前瞻规划能力

| 步骤 | 任务 | 说明 |
|------|------|------|
| 3.1 | **Look-ahead search** | 每个决策点，模型生成 "如果我这样做，接下来 3 步会怎样"，选择最优路径 |
| 3.2 | **MCTS + LM evaluation** | Monte Carlo Tree Search，用 LM 做 value estimation 和 policy prior |
| 3.3 | **Subgoal decomposition** | 模型显式输出子目标："这轮目标是攒钱买 Joker X"，"下一轮用 Flush 过 Boss" |
| 3.4 | **Multi-turn planning** | 从 Ante 1 的 shop 就开始规划 Ante 8 的 build path |

**成功标准**：Planning 模型 win rate >= 60%，且决策过程可解释

### Phase 4: Beyond Human Baseline

**目标**：在各种 stake / seed 条件下稳定超越人类水平

| 步骤 | 任务 | 说明 |
|------|------|------|
| 4.1 | **Multi-stake 泛化** | 在 Stake 1-8 上评测 |
| 4.2 | **Joker synergy 发现** | 模型是否能发现人类已知/未知的 Joker 组合 |
| 4.3 | **对比人类玩家** | 在相同 seed 上对比模型和高水平人类玩家的得分/通关率 |
| 4.4 | **可解释性报告** | 输出模型的"Balatro 攻略"——它学到了什么策略 |

## TOP 3（下一步立即执行）

### 1. Game State Text Serializer

把 engine snapshot 转成结构化文本，让 LM 能读懂游戏局面。

```
[STAGE] Blind - Small Blind
[SCORE] 180/300 (60% complete)
[RESOURCES] Plays: 2 | Discards: 1 | Money: $7
[HAND] 9S JC AD 6D JD 2H 5D 3H
[JOKERS] Joker (+4 Mult) | Greedy Joker (Diamonds +3 Mult)
[BOSS] None (Small Blind)
[ANALYSIS] Best hand: Pair of Jacks (JC+JD) ~60pts. Need 120 more.
```

### 2. Claude API Agent

用 Claude API 直接玩 Balatro，收集高质量 trajectory：
- 输入：text-serialized game state + legal actions
- 输出：CoT reasoning + chosen action
- 每局保存完整 (state, reasoning, action) 链
- 目标：100 局，筛选出 win rate > 30% 的优质 trajectory

### 3. LLM Evaluation Harness

自动化评测框架：
- 加载任意 LM（本地 23B 或 API）
- 跑 N 局（可指定 seed 列表）
- 输出：mean ante, win rate, mean score, 关键决策分析
- 对比不同模型 / 不同 checkpoint 的表现
